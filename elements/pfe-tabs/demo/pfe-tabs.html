Active Key is Disabled

Note:
If the [active-key] is set to a disabled tab index. The whole tablist is no longer focusable.

The PatternFly 4 implementation allows tab selection using the Tab key, however, that is counter to W3C tabs accessibility keyboard navigation practices which only allow the active tab to be focusable (in this case an non-focusable tab), then enabling keyboard navigation to select a different tab. The Tab key should then select the next element in the tab order. If a tab is active and its associated panel has a focusable item that item should be

If an aria-disabled tab is set as the [active-key] it still allows focus on the tablist enabling keyboard navigation.
