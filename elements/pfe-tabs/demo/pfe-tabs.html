Active Key is Disabled

Developer Note:

If the [active-key] is set to a [disabled] tab index. The tablist is no longer focusable.

The PatternFly 4 implementation allows tab navigation using the Tab key, however, that is counter to W3C aria tabs accessibility authoring practices which only allow the active tab to be focusable (in this case an non-focusable tab). This stops the user from entering the tablist via the Tab key.

If an [aria-disabled] tab is set as the [active-key] it still allows focus on the tablist enabling keyboard navigation via arrows and home and end keys.
